02 / Process

具体是怎么做的？

01

在飞书建文档，搭框架

打开飞书新建文档，先不想细节，只写大纲：培训目标是什么、分几个模块、每个模块解决什么问题。这一步完全是自己的业务判断，AI不介入——因为只有你最清楚店总们的实际痛点在哪里。

飞书文档 框架搭建 模块拆分 纯人工判断
02

借助 AI 工具，在飞书文档填充细节内容

框架确认后，并不是纯靠人工逐字填写——而是借助 DeepSeek、豆包等 AI 工具来辅助扩充细节：把框架标题丢给 AI，让它帮你补充场景描述、生成示例数据、完善注意事项。人工负责判断"写得对不对"，AI负责"把字填满"。这份文档既是内容的母版，也是后续喂给 Claude 的原材料。

飞书文档 DeepSeek 豆包 细节填充 场景扩写 结构化素材
03

把文档内容喂给 Claude，生成 HTML

把飞书文档内容复制进Claude，并给出明确指令。第一版HTML通常已经有80%的完成度，视觉效果远超预期——这一步是整个流程最"哇"的时刻。

实际使用的 Prompt 结构

角色：你是一位专业的前端设计师，擅长制作企业培训材料。

任务：根据以下培训手册内容，生成一个完整的HTML文件。
[粘贴飞书文档全部内容]

要求：
① 视觉风格：专业、现代、适合企业内训
② 结构：按模块分章节，有清晰导航
③ 核心内容：每个场景要有完整的prompt示例
④ 格式：单个HTML文件，无需外部依赖

Claude HTML生成 可交付成果
04

多轮对话迭代，打磨细节

第一版生成后，不要关闭对话。在同一对话窗口里持续提出修改要求，AI会记住上下文，每次只需要说"改什么"，不需要重新描述整个文档。这是提效的关键。

迭代追问示例

"第三章的prompt示例太长，压缩到150字以内"
"给每个模块加一个预计学习时间的标签"
"配色换成更沉稳的深色系，更适合企业风格"
"在结尾加一个'本周就要用'的行动承诺区"

Claude 多轮迭代 细节优化 最终版本
05

导出文件，部署到 GitHub，变成人人可访问的网页

将 Claude 生成的 HTML 代码保存为 `.html` 文件，上传到 GitHub 并开启 GitHub Pages，即可生成一个永久可访问的网页链接。把链接发到群里，店总们用手机扫一扫或点开链接，无需下载任何文件，随时随地浏览——这一步让培训手册从"一个文件"升级为"一个活的网页"。

部署流程（3步完成）

Step 1：注册 GitHub 账号，新建一个仓库（Repository）
Step 2：把 HTML 文件上传，文件名改为 index.html
Step 3：进入仓库 Settings → Pages → 选择 main 分支发布

完成后即获得形如 yourname.github.io/手册名 的永久链接

GitHub Pages 在线部署 链接分发 人人可浏览

使用注意事项

01

素材质量决定输出质量

飞书文档写得越具体，AI生成的内容越准确。模糊的框架只会得到模糊的结果。"给店总做培训"比"数据分析培训，目标是让店总3分钟内用AI生成周报摘要"差很多。

02

不要关闭对话窗口

整个制作过程保持在同一个Claude对话里完成。关闭窗口重开，AI就失去了对上下文的记忆，修改时需要重新说明背景，效率大幅下降。

03

人工把关是必须的

AI生成的内容需要人工审核，尤其是数字、流程、品牌名称等。不要盲目信任第一版，读一遍，用自己的业务判断做最终校验。AI是助手，判断权在你。

04 / Takeaway

带走这三件事

方法论

飞书写结构（你的业务判断） → Claude生成内容（AI的执行力） → 人工校验（你的专业负责）。三步缺一不可，顺序不能颠倒。

核心心态

AI是助理，不是替代者。它能把你从"格式和排版"里解放出来，让你把时间花在"判断和决策"上。这才是生产力提升的本质。
